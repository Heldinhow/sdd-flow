# Testing Infrastructure - SPEC

## User Stories

### US-1: High Coverage
As a **developer**, I want **>80% test coverage** so that **I can refactor confidently**.

### US-2: Fast Tests
As a **developer**, I want tests to **run in <30s** so that **feedback is quick**.

### US-3: Test Isolation
As a **developer**, I want **isolated tests** so that **results are consistent**.

## Requirements

### MUST
- [ ] Fix discoverCommands redundant calls (4x)
- [ ] Add tests for untested modules
- [ ] Improve test isolation

### SHOULD
- [ ] Add integration tests
- [ ] Add E2E smoke tests
