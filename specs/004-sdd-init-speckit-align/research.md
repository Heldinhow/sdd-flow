# Research: sdd-init Speckit Alignment — Deviation Audit

**Feature**: `004-sdd-init-speckit-align`
**Date**: 2026-03-22
**Purpose**: Catalogue every structural difference between `sdd-flow` and the native speckit
standard. Each deviation is classified as KEEP, REMOVE, or REVIEW.

---

## Reference: Speckit Standard Structure

The speckit native structure expects:

```
.specify/
  scripts/bash/    — workflow shell scripts
  templates/       — planning artifact templates
  memory/          — constitution storage
specs/
  <feature>/       — feature workspace (spec.md, plan.md, tasks.md, ...)
.opencode/
  command/         — speckit.* command definitions
```

No root-level files are installed by speckit itself. Feature workspaces may include
optional `contracts/` and `checklists/` subdirectories.

---

## Deviations Found

### ✅ KEEP — Intentional Product Decisions

These deviations are sdd-flow's value-add over raw speckit. They are not defects.

| Deviation | Where | Rationale |
|-----------|-------|-----------|
| `.opencode/skills/` directory with `sdd-flow`, `sdd-spec`, `sdd-plan`, `sdd-tasks` | `.opencode/managed-assets/.opencode/skills/` | The agent orchestration layer. Keeps planning behavior repo-owned rather than baked into hidden agent state. Intentional design. |
| `Spec Driven` unified agent (`sdd.md`) | `.opencode/command/sdd.md` | Auto-routes spec → clarify → plan → tasks so users don't manually invoke each `speckit.*` command. Core UX goal of the project. |
| `/sdd-init` bootstrap command | `.opencode/command/sdd-init.md` | Speckit has no equivalent onboarding step. Needed for non-destructive ADD/KEEP/REVIEW merge into existing repos. |
| `/implement` execution entrypoint | `.opencode/command/implement.md` | Speckit has no equivalent execution command. Loads planning artifacts and hands off to the build agent. |
| `.specify/memory/constitution.md` in managed-assets | `.opencode/managed-assets/.specify/memory/constitution.md` | Speckit supports constitution via `/speckit.constitution`. Bundling the template pre-populates it for target repos. Compatible — not a deviation. |

---

### ❌ REMOVE — Eliminated by This Feature

These deviations are being resolved in this implementation.

| Deviation | Where | Action |
|-----------|-------|--------|
| `AGENTS.md` at repo root in managed-assets | `.opencode/managed-assets/AGENTS.md` | **Deleted.** Pre-installing sdd-flow's own guidelines into unrelated repos was wrong. AGENTS.md is now generated organically during planning. |
| `MANAGED_ASSET_GROUP.GUIDE` asset group | `.opencode/src/init/managed-assets.ts` | **Removed.** The GUIDE group existed solely for AGENTS.md. Eliminated with it. |
| `hasAgentsFile` flag in `RepoState` | `.opencode/src/init/detect-repo-state.ts` | **Removed.** No longer needed once AGENTS.md is not a managed asset. |
| Phase 4.1 "Create AGENTS.md" in init checklist | `.opencode/command/sdd-init.md` | **Removed.** Init no longer creates or manages AGENTS.md. |
| "AGENTS.md is the primary development guidelines" rule in sdd.md | `.opencode/command/sdd.md` | **Replaced.** Constitution is now the governance document; agent files are generated by planning. |
| "Load AGENTS.md for active development guidelines" in speckit.analyze | `.opencode/command/speckit.analyze.md` | **Updated.** AGENTS.md is now loaded if present (opportunistic), not required. |

### 🔍 REVIEW — Out of Scope, Needs Future Decision

These items warrant attention but are not addressed in this feature.

| Deviation | Where | Notes |
|-----------|-------|-------|
| `update-agent-context.sh` maps Amp, Kiro CLI, IBM Bob, Pi → `AGENTS.md` | `.specify/scripts/bash/update-agent-context.sh` | AGENTS.md is created organically here during planning, not pre-installed. Acceptable for now. However, the agent mappings should be validated against each agent's current naming conventions (some agents may now use different filenames). |

## Summary

After this feature:
- The managed-assets bundle installs **only** files under `.specify/`, `.opencode/`, and `specs/`
- No root-level files are pre-installed by `/sdd-init`
- The folder structure is fully aligned with the speckit standard
- One intentional deviation class remains: the `.opencode/skills/` orchestration layer,
  which is sdd-flow's primary differentiator from raw speckit and should be kept
