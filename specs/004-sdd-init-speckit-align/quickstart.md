# Quickstart: Verify sdd-init Speckit Alignment

**Feature**: `004-sdd-init-speckit-align`
**Date**: 2026-03-22

## Verification Steps

### 1. TypeScript passes with no errors

```bash
cd .opencode && bunx tsc --noEmit
# Expected: no output (clean)
```

### 2. Unit tests pass

```bash
cd .opencode && bun test tests/unit/packaging/ tests/unit/init/
# Expected: all tests pass, 0 fail
```

### 3. AGENTS.md is absent from the managed-assets bundle

```bash
ls .opencode/managed-assets/
# Expected: only .opencode/ and .specify/ directories — no AGENTS.md
```

### 4. No unexpected AGENTS.md references remain in the bundle

```bash
grep -r "AGENTS.md" .opencode/managed-assets/ | grep -v "update-agent-context"
# Expected: only the opportunistic-load line in speckit.analyze.md
# That line reads: "If `AGENTS.md` exists in the repository root..."
```

### 5. Manifest no longer includes AGENTS.md

The `buildManagedAssetManifest` function now only collects files under `.specify/`
and `.opencode/`. Verify by running the init unit test:

```bash
cd .opencode && bun test tests/unit/init/repo-init.test.ts
# Expected: "builds a managed asset manifest" test passes and asserts AGENTS.md is absent
```
